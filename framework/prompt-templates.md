# Prompt Templates

Use these as starting points. Replace [brackets] with company-specific language.

---

## Category Prompts
*"Best X for Y" — where naming a specific tool is a natural response.*

- best [product category] software for [industry/use case]
- best [product category] platform for [company type] in [geography]
- top [product category] tools for [job title]
- best AI platform for [specific workflow]
- [product category] software comparison [year]

---

## Competitive Prompts
*Direct comparison or alternatives — highest commercial intent.*

- alternatives to [Competitor A]
- [Company] vs [Competitor B]
- [Competitor A] competitors
- why companies switch from [Competitor A]
- [Company] alternative for [specific use case]

---

## Problem Prompts
*Buyer describes a pain point — tool recommendation is a natural next step.*

- how to [solve specific operational problem]
- best way to [reduce/improve/automate] [specific metric]
- how do [industry] companies [solve problem]
- what software helps with [specific pain point]
- how to [job to be done] without [current broken method]

---

## Emerging Prompts
*Category-defining questions that no one owns yet.*

- what is [new category name]
- how does [new approach] work for [industry]
- [new technology] for [legacy workflow]

---

## Prompt Hygiene Rules

1. If a specific brand name is NOT a natural response, remove the prompt.
2. Use language buyers use, not how the company describes itself.
3. Include geography when it matters (India, US, EU).
4. Include buyer role when it sharpens the prompt ("for a DISCOM CTO", "for a manufacturing ops head").
5. Keep prompts under 15 words where possible — longer prompts produce generic answers.
